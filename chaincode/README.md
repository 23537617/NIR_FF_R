# Смарт-контракт TaskDocument (TypeScript)

Этот репозиторий содержит смарт-контракт (чейнкод) `taskdocument` для сети Hyperledger Fabric, написанный на **TypeScript**.

Чейнкод отвечает за управление документами-задачами (Task Documents) внутри распределенного реестра (Ledger) и реализует бизнес-логику для хранения, проверки и отслеживания состояния задач в защищенной блокчейн-сети.

## 🏗 Архитектура разработки

Чейнкод написан с использованием официального **TypeScript SDK** (`fabric-contract-api` и `fabric-shim`), что обеспечивает:
- **Строгую типизацию**: Все структуры данных проверяются на этапе компиляции.
- **Декларативное описание транзакций**: Классы смарт-контрактов размечены декораторами `@Info`, `@Transaction`, `@Returns()`.

#### 📦 Развертывание (Chaincode-As-A-Service)
Для удобства разработки, отладки и обновления чейнкод развертывается в сети Fabric как внешний сервис (**CCAAS**), а не встраивается внутрь пира. 
Скрипт `deploy_chaincode.py` привязывает смарт-контракт к Docker-контейнеру `chaincode-server`, в котором выполняется данный код.

## ✨ Реализованный функционал (Advanced Features)

Помимо базовых операций CRUD, в смарт-контракте задействован расширенный функционал Hyperledger Fabric:

1. 🛡 **Client Identity (Идентификация клиента)**
   При создании задачи автоматически определяется и записывается создатель (MSP ID и данные сертификата X.509) с помощью `ctx.clientIdentity`.
2. 📜 **History API (Слепок истории транзакций)**
   Метод `get_task_history(task_id)` позволяет получить весь жизненный цикл конкретной задачи — от ее создания до последнего изменения статуса, включая точные метки времени (`tx_timestamp`) и идентификаторы транзакций.
3. 🔍 **Rich Query (Предметно-реляционные запросы)**
   Так как в сети используется CouchDB, чейнкод поддерживает "богатые запросы". Метод `query_tasks` принимает JSON-селекторы CouchDB (например, `{"selector":{"docType":"task","assignee":"Ivanov"}}`), позволяя искать задачи не только по ключу, но и по значениям полей.
4. 📑 **Pagination (Пагинация выборок)**
   Для работы с большими объемами данных `query_tasks` поддерживает безопасную пагинацию (передаются аргументы `page_size` и `bookmark`). 
5. 🔔 **Events (События чейнкода)**
   Каждая мутирующая транзакция (например, `create_task`, `update_task_status`, `add_document_version`) генерирует блокчейн-событие (`onTaskCreated`, `onTaskStatusUpdated`). Внешние приложения могут подписываться на эти события для мгновенного обновления интерфейса.
6. ⏱ **Детерминированность**
   В смарт-контракте не применяется системное время (`new Date()`). Вместо него жестко используется консенсусное время транзакции из Fabric (`ctx.stub.getTxTimestamp()`), что гарантирует идентичность результатов (Read-Write Set) на всех пирах.

## 🚀 Деплой и запуск

Поскольку используется архитектура **CCAAS**, запуск состоит из двух этапов: запуска контейнера с логикой и регистрации чейнкода в сети.

### 1. Запуск контейнера чейнкода (сервера TypeScript)
В директории с чейнкодом необходимо собрать и запустить Docker-контейнер:
```bash
docker compose -f docker-compose.chaincode.yaml up -d --build chaincode-server
```
Это соберет TypeScript-проект и поднимет сервер, слушающий порт `9999`.

### 2. Регистрация в сети Fabric
Если сеть разворачивается с нуля, нужно сообщить пирам, где находится сервер чейнкода. Для этого используется утилита:
```bash
python deploy_chaincode.py
```
Скрипт создаст архив с `connection.json`, установит его на все пиры в обеих организациях и выполнит коммит (Commit) чейнкода в канал.

*Примечание: При изменении только бизнес-логики (`taskContract.ts`) заново выполнять `deploy_chaincode.py` не нужно. Достаточно перезапустить Docker-контейнер `chaincode-server`, и сеть Fabric начнет использовать обновленный код.*

## 🛠 Структура проекта (папка `chaincode-ts`)

- `src/index.ts`: Главный входной файл чейнкода, экспортирующий контракты.
- `src/models.ts`: TypeScript-интерфейсы `Task` и `DocumentVersion`, описывающие структуру сохраняемых данных.
- `src/taskContract.ts`: Основная логика выполнения транзакций.
- `Dockerfile`: Мульти-этапная сборка контейнера, компилирующая TypeScript в JavaScript.
- `package.json`: Зависимости и скрипты сборки/запуска.

## 📄 Обзор API (Основные транзакции)

* `create_task(task_id, title, description, assignee)` — Создает новую задачу, присваивая ей статус `NEW`.
* `get_task(task_id)` — Возвращает текущее состояние задачи.
* `update_task_status(task_id, new_status, updated_by)` — Изменяет статус задачи и генерирует соответствующее событие.
* `get_task_history(task_id)` — Возвращает историю всех изменений задачи.
* `query_tasks(query_string, page_size, bookmark)` — Поиск задач с помощью CouchDB-запросов.
* `add_document_version(task_id, document_id, version, content_hash, uploaded_by, metadata)` — Добавляет ссылку на версию документа с привязкой к конкретной задаче.
* `get_document_versions(task_id, document_id)` — Возвращает историю версий для указанного документа.
